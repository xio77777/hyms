# 当前工作上下文

## 当前工作重点
无障碍优化第二轮 - 修复交互缺陷，降低运动控制要求，支持遥控器操作

## 最近变更
- 2026-07-08: 第二轮第一性原理优化，修复关键交互问题：

### 计时器下拉菜单修复
- **修复 bug**：计时器选项下拉菜单之前始终显示，造成认知混乱
- 改为点击切换显示/隐藏，选择后自动关闭

### 速度控制简化
- **滑块改为预设按钮**：脑梗患者手部精细运动困难，滑块需要精确拖拽
- 改为三个预设按钮：慢(0.5x)、中(1.0x)、快(1.5x)，一键切换
- 每次切换有语音反馈

### 无障碍设置全局化
- **修复缺陷**：highContrast/largeFont 之前仅在首页生效
- 改为在 App.tsx 根组件通过 useEffect 在 document.documentElement 上添加/移除 class
- 所有页面统一生效，CSS 全局样式覆盖

### SensoryTraining 页面统一
- **修复不一致**：SensoryTraining 仍是旧版小按钮(px-3 py-2, w-5 h-5)
- 统一为与其他训练页面一致的大按钮(px-5 py-3, w-6 h-6)
- 添加语音反馈（返回首页）
- 标题字号从 text-sm 增大到 text-lg

### 键盘/遥控器导航支持
- **新增 useKeyboardNav hook**：支持方向键在可聚焦元素间导航
- 方向键(上下左右)：在按钮间循环聚焦
- Escape：关闭弹出菜单或返回首页
- 适配大屏电视遥控器操作场景

## 活跃决策与考虑
- **运动控制最小化**：脑梗患者偏瘫侧手部精细操作困难，所有交互应支持一键操作
- **遥控器优先**：大屏电视是主要使用场景，遥控器方向键导航是核心需求
- **全局一致性**：无障碍设置必须在所有页面生效，不能只在首页

## 重要模式与经验
- 滑块控件对运动障碍患者不友好，预设按钮更合适
- 无障碍设置必须在应用根级别全局应用，不能依赖各页面自行处理
- 键盘导航 hook 应过滤不可见元素，避免焦点丢失
- TypeScript 5.8.x 的 ignoreDeprecations 值为 "5.0"
