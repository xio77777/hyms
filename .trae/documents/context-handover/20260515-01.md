## 4. 已确认的决策

- 决策：使用 React Portal 解决投屏弹窗定位问题
- 原因：ControlBar 使用 backdrop-blur 创建了新的包含块，导致弹窗 fixed 定位失效
- 状态：已实现并测试

- 决策：使用 MediaRouter + Presentation API 实现投屏
- 原因：Android 原生方案，不依赖第三方服务，支持 Miracast
- 状态：已实现

- 决策：投屏使用本地资源路径而非 localhost
- 原因：电视端 WebView 无法加载 localhost 地址
- 状态：已修复

- 决策：移除 vite-plugin-trae-solo-badge
- 原因：用户不需要右下角 TRAE SOLO 徽章
- 状态：已移除

## 5. 历史事故和硬约束

事故1：投屏弹窗始终显示在底部，不在屏幕中央
- 症状：点击投屏按钮，弹窗显示在底部控制栏区域上方，不居中
- 根因：ControlBar 使用 backdrop-blur 创建了新的 CSS 包含块，弹窗在 ControlBar 内部使用 fixed 定位时相对于 ControlBar 而非视口
- 预防规则：使用 backdrop-blur 时，其子组件不要使用 fixed 定位，或使用 Portal 渲染到 body 上

事故2：投屏后电视端黑屏/无内容
- 症状：连接到电视后，电视端 WebView 无法加载内容
- 根因：CastPlugin 构建的 URL 是 https://localhost，电视端无法解析
- 预防规则：电视端 WebView 必须使用 file:///android_asset/ 本地资源路径
